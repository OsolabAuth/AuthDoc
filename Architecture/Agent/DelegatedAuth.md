# AIエージェント委譲認証設計

## 目的

人間ユーザーの通常トークンをAIエージェントへ渡さず、ユーザーが明示的に委譲した範囲だけをAIエージェントが代理実行できるようにする。

## 方針

- `sub` はAIエージェント自身のIDにする。
- 代理元ユーザーは `owner_sub` として表す。
- `agent_id` / `agent_secret` で短命トークンを発行する。
- scope、client_id、有効期限で委譲範囲を制限する。
- 作成、発行、利用、失効を監査ログに記録する。

## 初期スコープ

- task.read
- task.create
- task.comment
